Object Oriented Programming Part 1

What is OOP?

OOP stands for Object Oriented Programming. It is a type of computer programming that lets the programmer define not only the data type of a data structure, but also its functions. An object in programming is similar to an object in real life in the way that they both have properties and behaviors. An dog will be a set color (brown, black, etc), and will also have behaviors, (run, jump, etc). A dog object in java will first have to be defined in a class. This class is essentially a list of attributes and behaviors of an object. In java, an attribute is called a variable, and a behavior is called a method. So the class "dog" will have all the attributes and behaviors that you want a dog object to have. When you create the dog object, it will have all of the attributes and behaviors of its class.

 
/*Example dog class

Principles of OOP

There are three fundamental principles of OOP.
Abstraction, which lets the programmer define the basic outline of a class, Encapsulation, which lets the programmer hide data, and inheritance, which lets the programmer make subclasses have inherit properties from a superclass. An abstract class is a class where there is at least one method without implementation. This is useful if you just want to have a very basic template for your subclasses.

//Example of abstract class
abstract class Teacher{  
    abstract void teach();  
}  
                    
//class that provides implementation for teacher
class BillNye extends Teacher{
    teach(){
        System.out.println("Science rules");
    }
}

Encapsulation is useful for Java programmers when they want to keep data separate between classes. It lets the programmer control where the data goes, which makes your code much easier to organize, among other thing. 

//we can hide data by using the "private" access modifier
public class Dog{  

    //private data member  
    private String name;  
    
    //getter method for name  
    public String getName(){  
        return name;  
    }  
    
    //setter method for name  
    public void setName(String name){  
        this.name=name  
    }  
}  

//This is the class used to demo the methods
class Demo{  
    public static void main(String[] args){  
        //creating instance of the encapsulated class  
        Dog d =new Dog();  
        //setting value in the name member  
        d.setName("Buddy");  
        //getting value of the name member  
        System.out.println(d.getName());  
    }  
}

In java, classes can be many different things. Two of the things they can be are superclasses and subclasses. A superclass is a class that is being inherited from. Its methods and variables will be inherited into all of its subclasses, unless the programmer specifies otherwise.
A subclass is a class that has inherited properties from a superclass. The superclass's variables and methods can be called(used),in a subclass without it being written in the subclass's code.

//This is the superclass
class SuperClass{
    
    public void superClassMethod{
        System.out.println("I am from a superclass");
    }
}

//the subclass will inherit the superClassMethod from the superclass
class SubClass extends Superclass{
    public void subClassMethod{
        System.out.println("I am from a subclass");
    }
}

class demo{
    public static void main (String[] args){
        SubClass s = new SubClass;
        s.superClassMethod();
        s.subClassMethod();
    }
}

Polymorphism

Polymorphism is another part of object oriented programming. In programming terms, it means "The ability of an object to take many forms". A common way of using polymorphism is when you use a parent class reference to refer to a child class reference. In other words, you can use a superclass reference to refer to a subclass reference.

//This is the superclass
class SuperClass{
    
    public void superClassMethod{
        System.out.println("I am from a superclass");
    }
}

//the subclass will inherit the superClassMethod from the superclass
class SubClass extends Superclass{
    public void subClassMethod{
        System.out.println("I am from a subclass");
    }
}

class demo{
    public static void main (String[] args){
        SubClass s = new SuperClass;
        //this code will work because the subclass 
        //will automatically upcast to become a superclass
    }
}

Arraylists are a list of arrays, so we can say that they are an array of arrays. They are different than multiple dimensional arrays, because of how they are a dynamic data structure. Normal arrays are a static data structure, so they are stuck to the initial size that you give them. If you want to make an array larger, you have to copy the contents to a new array, and initialize that array to a larger size. An arraylist can be made larger or smaller. By using the arraylist.add() and arraylist.remove() methods, we can add or remove elements from the arraylist. This is useful when you want a dynamic list of something that doesn't have a set amount of elements. For example, arraylists are more useful if you want to create a sign in sheet for an event that you don't know the number of addendants. The arraylist will resize automatically as you keep adding the attendants.

Heap Sort

Heap sort is a sort based off of the binary tree based data structure, heaps. There are two types of heaps, max heaps and min heaps, but for our tutorial we will be using max heaps. As stated before, heaps are based off of binary tree data structures, which basically look like this:

As you can see, there are root nodes, nodes and leaves. A tree will only have one root node, and it is typically the first index of the array. Each node can only have a maximum of two child nodes. We will refer to leaf nodes as just nodes, because it doesn't make a difference in the explanation. Max heaps are essentially trees that meet a certain condition. To be a max heap, the tree's root node must be larger or equal to its child nodes, and its child nodes must be greater or equal to their child nodes, and so on and so forth. Here we can see an example of a max heap:

Trees are made from arrays or arraylists, and the root nodes are typically the first index. The left child node is then set to the second index, and the right node set to the third. After creating our tree, we will need to change it into a max heap, so the parent nodes must be larger or equal to its child nodes. We will call this method heapify. Here is the implementation to heapify an array. 

    // To heapify a subtree rooted with node i which is 
    // an index in arr[]. n is size of heap 
    void heapify(int arr[], int n, int i) 
    { 
        int largest = i; // Initialize largest as root 
        int l = 2*i + 1; // left = 2*i + 1 
        int r = 2*i + 2; // right = 2*i + 2 
  
        // If left child is larger than root 
        if (l > n && arr[l] > arr[largest]) 
            largest = l; 
  
        // If right child is larger than largest so far 
        if (r > n && arr[r] > arr[largest]) 
            largest = r; 
  
        // If largest is not root 
        if (largest != i) 
        { 
            int swap = arr[i]; 
            arr[i] = arr[largest]; 
            arr[largest] = swap; 
  
            // Recursively heapify the affected sub-tree 
            heapify(arr, n, largest); 
        } 
    }
